# 服务器缓存清理和重启指南

## 问题背景
在开发过程中，Next.js 服务器可能会出现缓存问题，导致页面显示异常、样式错乱或页面重叠等问题。这通常是由于 `.next` 缓存目录中的旧文件导致的。

## 解决方案

### 方法1：使用批处理文件（推荐）
双击运行项目根目录下的 `clean-restart.bat` 文件，它会自动：
1. 停止所有正在运行的 node 进程
2. 清理 `.next` 缓存目录
3. 清理 `node_modules/.cache` 目录
4. 在新窗口中重启开发服务器

### 方法2：使用 PowerShell 脚本
在项目根目录下运行：
```powershell
.\scripts\clean-restart.ps1
```

### 方法3：使用 npm 脚本
现在可以通过 npm 脚本更方便地清理和重启：
```bash
npm run dev:clean
```
这个脚本会执行与批处理文件相同的清理步骤，并在新窗口中启动开发服务器。

### 方法4：手动清理
如果您想手动清理，可以执行以下步骤：

1. **停止服务器**：
   ```powershell
   # PowerShell
   Stop-Process -Name "node" -Force
   
   # 或使用任务管理器
   taskkill /F /IM node.exe
   ```

2. **清理缓存目录**：
   ```powershell
   # 清理 .next 目录
   Remove-Item -Recurse -Force ".next"
   
   # 清理 node_modules/.cache 目录
   Remove-Item -Recurse -Force "node_modules\.cache"
   ```

3. **重启服务器**：
   ```powershell
   npm run dev
   ```

## 常见问题

### Q: 为什么需要清理缓存？
A: Next.js 在开发模式下会缓存构建文件以提高性能。但有时缓存会变得过时或损坏，导致页面显示问题。

### Q: 清理缓存会删除我的代码吗？
A: 不会。只会删除 `.next` 目录中的构建缓存文件，您的源代码不会受到影响。

### Q: 清理后需要重新安装依赖吗？
A: 不需要。`node_modules` 目录不会被删除（除了 `.cache` 子目录）。

### Q: 每次修改代码后都需要清理吗？
A: 通常不需要。只有在遇到以下问题时才需要清理：
- 页面样式错乱
- 页面元素重叠
- 修改了代码但页面没有更新
- 服务器启动失败

## 自动化建议

为了确保每次任务完成后都能自动清理缓存，建议：

1. **在完成重要修改后**运行清理脚本
2. **在切换分支前**运行清理脚本
3. **在遇到页面显示问题时**首先尝试清理缓存

## npm 脚本使用说明

现在项目提供了 `dev:clean` npm 脚本，可以更方便地清理和重启服务器：

```bash
npm run dev:clean
```

### 脚本功能
- 自动停止所有 Node.js 进程
- 清理 `.next` 构建缓存
- 清理 `node_modules/.cache` 缓存
- 在新窗口中启动开发服务器

### 使用建议
- 如果遇到缓存问题，建议使用此脚本代替 `npm run dev`
- 可以在每次开发会话开始时运行此脚本，确保从干净状态启动
- 脚本会保留您的源代码和依赖，只清理缓存文件
